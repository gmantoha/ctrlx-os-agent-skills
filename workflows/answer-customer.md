# Answer Customer

Use this workflow when the primary output is a technical answer to a customer or colleague.

## Rules

- Separate verified facts from interpretation.
- Prefer platform-native approaches over clever workarounds.
- Include examples, commands, UI paths, or code when they make the answer more actionable.
- Use labs or devices for verification when helpful and safe.
- Cite the local reference, case, PDF, or direct observation that supports the answer.

## Standard Flow

1. Identify the customer question and product/app scope.
2. Read relevant `reference/` content and cases.
3. Verify against docs, PDFs, OpenAPI descriptions, or a lab/device when practical.
4. Draft a concise answer with assumptions and caveats clearly marked.
5. Include reproducible steps or examples if the user can act on them.

## Template

Use `templates/customer-answer-template.md` when a structured customer response is useful.
